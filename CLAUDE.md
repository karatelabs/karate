Start with [docs/DESIGN.md](./docs/DESIGN.md) — the primary architecture reference for the codebase (and the deep-dive docs it links).

## Conventions

- **No issue/PR numbers in source.** Describe the behavior or rationale instead of citing a GitHub issue — in comments, test names, and Gherkin `Feature:`/`Scenario:` names. Commit messages (`fixes #123`) and load-bearing third-party tracker links are fine. Not a hard failure, just the house style.
